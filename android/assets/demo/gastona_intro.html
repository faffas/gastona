What is Gastona

Gastona is a scripting language to make applications easily. Among other features it has "out of the box" things like

  • rapid GUI building
  • SQL integrated
  • procedures like scaning files and parsing text
  • communications using UDP and an amazing HTTP server integrated!

All are usually considered advanced and complex subjects but using them with Gastona is really straighforward. This is an important point since they are very powerful tools to build applications and hopefuly Gastona will help to demistify them.

Gastona is an open source project implemented in java and it has two variants

  • Desktop/PC (gastona.jar) that runs in Windows, Linux, Raspberry Pi and Mac OSX
  • Android App (gastona.apk) for android smart phones and tablets

Both use exactly the same script language so it is possible to develop an application that will run in any of these systems.

Is Gastona different ?

Yes, it is. It has no similarity to any programming language. The main goal is precisely to save programming effort as much as possible.

This can be achieved with a very pragmatic approach, simplification if you want, like to say "a button is a button". And also very important, using an unsophisticated but flexible enough data structure for all purposes of the language.

The best way to learn it is to run the desktop application (gastona.jar) which comes with a complete documentation of the language, widgets and commands. Almost every element in the documentation has one or several related sample scripts that can be modified and run immediately.

Getting Gastona

The App for Android is available at the Google market for aproximately 1 Euro which represents a support for the project. It is also possible to download it for free and install it manually but then, please, be sure to get gastona.apk from the official site or some source you trust!

Binaries for both desktop and android can be found at sourceforge.net/projects/gastona/. If you want to contribute to its development or improvement or just build a binary from the code feel free to fork the project from github.com/wakeupthecat/gastona

Gastona on Android

The gastona app is prepared to open files with the termination ".gast" so if you copy your script in the sdcard of your device. A proper file explorer should ask you to open it with Gastona and give you the possibility to open it always with it. This is a standard functionality in Android.

Beside this, if the App is launched from the Apps panel, that is without providing a particular script then a default script called autoStart.gast wich uses other pure gastona scripts will be oppened. The default script has following

  • microTutorial A list of some minimalistic samples to be run
  • File Ciervo This script is a file server, so you can examine and download files of your device from a browser in your PC
  • programMe you can see, modify, execute and save scripts on your device from a browser in another device (i.e. PC)
  • Gast explorer Explore and launch gast files located at /mnt/sdcard/gastona folder

Both programMe and Gast explorer are limited to /mnt/sdcard/gastona and subfolders for convenience. Note that this folder, if created, does not belong to the App so for example, it will be not removed if the App is uninstalled.

If you want to examine these initial scripts they are all available (depending on the version) at master/android/assets

Both scripts File Ciervo and programMe use the mini HTTP server included in gastona (command MICOHTTP). They are very good examples of the power of this feature to communicate devices.

File Ciervo

This, less than 200 lines script is an amazing File server. With it you can explore from your PC all files of your android device and also copy them to your PC (download).

This script acts as a small http server so a connection, for example via WiFi, between your PC and your android device is needed. Also you need to know the IP address or the host name that the android device has in your network. Once you have all this:

  • Launch File Ciervo on android
  • Open a browser in your PC and introduce the url http://(android ip):7878

Now the interesting part actually occurs in your browser where you can navigate through your android file structure. On your device you will see lost of messages. That just indicates the activity of the http command involved and it might be useful to check if the connection is doing well.

Do you need the contrary direction ? That is, serving files from yout PC to your device ? Guess it: run FileCiervo.gast in your PC using gastona.jar, done! ;)

ProgramMe

This script also connects your android device with your PC in a similar way to File Ciervo. This time oriented to handle and transfer your gastona scripts from and to your android device! Once you have running ProgramMe on your android device connect to it from your browser using the port 7676, then you can

  • Load scripts from the android device into the browser (PC)
  • Edit scripts directly in the browser
  • Run scripts into the device from the browser
  • Save scripts into the device from the browser

That is, almost full controll of your scripts using a browser.

About obtaining the IP address

Depending on the network configuration it might be more or less difficult to get it. If available, the host name of the device in the network is more convenient than the IP since it will not change so you can store the final url in your browser favorites and connect the next time just with a click.
